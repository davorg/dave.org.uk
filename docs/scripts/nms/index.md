This is pure nostalgia.

Back around the turn of the millennium, people started experimenting with
CGI programs on their websites. Everyone wanted a guest book, a hit counter
or (worst of all) a formmail.

And Matt Wright was happy to supply these programs. His "Matt's Script
Archive" became one of the most popular sites on the early World Wide Web.
People would download his programs and install them on their websites.

And open themselves to a world of pain.

You see, Matt wasn't really a programmer. He certainly wasn't a Perl
programmer. He just took a set of programs that had been written by
Reuven Learner, tweaked them a bit and made them available from his website.
It turned out that Matt was better at what passed for marketting in those
days and it was his versions of the programs that became insanely popular.

This led to two problems:

* The early web was a very different place. And as it transformed into the
rather unpleasant place we know today, major security holes in Matt's code
became obvious.
* People mistook popular code for good code. Many people learned Perl from
reading Matt and Reuven's rather dodgy code. Which meant that badly-written
and insecure code became very common on websites.

The [London Perl Mongers](https://londonperl.org/) decided to do something
about this. We started a project to rewrite all of Matt's programs in
high-quality Perl and with all of the security holes fixed. That project
was called [nms](https://nms-cgi.sf.net/) (yes, that originally stood for
"Not Matt's Scripts" - but very early on we decided that was too personal
and from then on "nms" stood for nothing).

And the project became pretty popular. I remember spending months trawling
internet forums, finding people who were looking for software recommendations
and suggesting that they used our code instead of Matt's. I started seeing
other people making the same recommendation and decided the project was a
success.

And then I found that Matt had even added a page to his site
[talking (positively) about nms](https://www.scriptarchive.com/nms.html).
We never took his site down though - I guess all those ads are pretty
lucractive :-)

It all seems so long ago though. Our code hasn't been updated for years
(probably decades). There are far better ways to add things like that to
a website these days (hmmm... maybe that would make a good blog post).

Our code is still [available online](https://nms-cgi.sf.net/), but it's
really only there for historical reasons. I'd add a disclaimer to the
top of the page, but I'd need to remember my Sourceforge credentials
in order to edit the site.
